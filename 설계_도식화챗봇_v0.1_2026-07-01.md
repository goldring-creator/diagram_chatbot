# 도식화 챗봇 — 설계 문서 (v0.1 · 2026-07-01)

> 목적: 연구자가 글이나 문서(PDF·Word·txt)를 넣으면, AI가 내용을 분석해 사회과학
> 논문용 학술 도식(분석틀·경로모형·트리·플로우차트·레이더·정책흐름 시계열)을
> 자동으로 그려 주는 데스크톱 챗봇을 만든다.
> 이 문서는 코드 작성 전 '무엇을·어떻게 만들지'를 확정하는 설계도다.

---

## 1. 한 줄 정의

문서를 넣으면 → AI가 도식 '설계도(JSON 스펙)'를 뽑고 → 파이썬 렌더러가 공식으로
좌표를 계산해 → 무채색 학술 SVG/PNG 그림을 그려 주는 챗봇.

## 2. 왜 만드는가 (기존 '연구설계 챗봇'과의 관계)

- 이미 `H:\...\브랜딩-코딩\연구설계 챗봇\소스코드`에 검증된 챗봇 인프라가 있다.
  NVIDIA 무료 API 호출, 사용자 키 관리, 파일 읽기(PDF·docx·txt·md)·OCR,
  한자·가나 오염 검사, Tkinter UI, 스트리밍, 내보내기가 모두 구현되어 있다.
- '도식화 챗봇'은 이 인프라의 **형제 앱(fork)**이다. 두뇌(prompts)와 출력(그림)만
  다르고, 골격(core·UI 셸)은 최대한 재사용한다.
- 결정적 차이: 기존 앱은 **글**을 출력하고, 이 앱은 **그림(SVG/PNG)**을 출력한다.

## 3. 품질 기준 (참고 그림에서 도출)

기준 폴더: `H:\...\with 전재은 교수님\3. Paper\38. 그림`
(그림1 분석틀, 그림3 공공성 6차원 레이더, 그림4 정책흐름 수렴 시계열 등)

관찰된 '집안 스타일(house style)' — 렌더러가 무조건 지켜야 할 규격:

| 항목 | 규격 |
|------|------|
| 색 | 무채색(그레이스케일)만. 채도 있는 색 금지. |
| 팔레트 | 배경 `#fff`, 강조 박스 `#333`/`#222`(흰 글씨), 일반 박스 `#e9e9e9`·`#f2f2f2`, 테두리 `#555`·`#888`·`#ddd`, 흐린 텍스트 `#777`·`#999` |
| 글꼴 | `'Apple SD Gothic Neo','AppleGothic','Malgun Gothic',sans-serif` (윈도우 대응 위해 맑은 고딕 추가) |
| 강조 박스 | `rx=9~11`, 제목 16~18px 굵게, 부제 11~12px 흐린색 |
| 화살표 | `marker` 삼각형, 주 경로 `stroke-width` 1.6~1.8 실선, 부 경로 점선 |
| 구역/보조 박스 | `stroke-dasharray="4,3"` 점선, 연한 채움 |
| 캔버스 | 가로형 `viewBox` 약 960~980 × 560~575 |
| 각주 | 하단에 `※`로 시작하는 10px 흐린 안내문 |
| 문자 | 한자·가나 절대 금지(생성 후 자동 검사, 전역 규칙) |

`svg-academic-diagram` 스킬(`~/.claude/skills/svg-academic-diagram`)의 좌표 공식과
대칭 검증 체크리스트를 렌더러 코드로 그대로 이식한다. 즉 **감각이 아니라 공식으로
배치하고, 추정이 아니라 계산으로 검증한다.**

## 4. 핵심 아키텍처 — 2단계 (스펙 → 렌더러)

사용자 확정: AI가 SVG를 통째로 쓰지 않는다. 대신 아래 2단계로 나눈다.

```
[문서/글 입력]
   │  (core: 파일읽기·OCR)
   ▼
① 이해·설계 (AI, NVIDIA API)
   - 문서 내용 분석 → 어떤 도식이 적합한지 판정
   - 도식 '설계도'를 정해진 JSON 스키마로만 출력 (그림을 직접 그리지 않음)
   ▼
② 렌더링 (파이썬 렌더러, AI 없음)
   - JSON 스펙 검증(스키마·필수필드·한자검사)
   - svg-academic-diagram 공식으로 좌표 계산 → 대칭 자동 검증
   - 무채색 집안 스타일 SVG 생성 → (PNG 변환)
   ▼
[SVG/PNG 그림 + 미리보기]
```

이 구조의 장점:
- **대칭·정렬이 코드로 보장**된다(LLM의 미세 비대칭 실수 원천 차단).
- **집안 스타일(무채색·글꼴·크기)이 코드에 고정**되어 그림마다 일관.
- **한자·가나 오염을 렌더 직전에 확정적으로 차단**.
- AI 출력이 짧고 구조적(JSON)이라 재현성·수정이 쉽다(노드 하나만 고쳐 재렌더).

### 4-1. AI가 출력하는 도식 스펙 스키마(JSON)

```jsonc
{
  "diagram_type": "framework | path_model | tree | flowchart | radar | timeline",
  "title": "그림 제목(선택)",
  "caption": "하단 ※ 각주(선택)",
  "nodes": [
    { "id": "n1", "label": "유보통합", "sub": "(분석 대상)",
      "role": "emphasis | normal | ghost", "group": "정책" }
  ],
  "edges": [
    { "from": "root", "to": "n1", "style": "solid | dashed", "label": "" }
  ],
  // 유형별 추가 필드 (렌더러가 유형에 맞게만 읽음)
  "levels": [ ["root"], ["n1","n2","n3"] ],        // tree
  "axes": ["접근성","공적관리","공적비용","공적지식","참여성","공익"], // radar
  "series": [ { "name":"유보통합","values":[3,3,3,2,3,3] } ],           // radar
  "timeline": { "start":2018, "end":2025,
                "lanes":["문제 흐름","정책 흐름","정치 흐름"],
                "events":[ {"lane":0,"year":2018,"label":"...","weight":"강"} ] }
}
```

- AI는 **오직 이 JSON만** 코드블록 없이 출력하도록 시스템 프롬프트로 강제한다.
- 렌더러는 알 수 없는 필드는 무시하고, 필수 필드 누락 시 사용자에게 되묻는다.

### 4-2. 렌더러가 지원하는 도식 유형(우선순위)

| 순위 | 유형 | 대응 참고그림 | 렌더 공식 출처 |
|------|------|---------------|----------------|
| 1 | `framework` 분석틀(상위명분→병렬대상→축 매트릭스) | 그림1 분석틀 | 트리+매트릭스 혼합 |
| 2 | `tree` 계층·분류 | 스킬 B | 스킬 트리 공식 |
| 3 | `flowchart` 순서·의사결정 | 스킬 C | 스킬 플로우차트 공식 |
| 4 | `path_model` 경로모형(LGM·SEM·매개) | 스킬 A | 스킬 경로모형 공식 |
| 5 | `radar` 다차원 프로파일 | 그림3 레이더 | 극좌표 등각 배치 |
| 6 | `timeline` 다중흐름 시계열 | 그림4 정책흐름 | 레인별 연도축 배치 |

v0.1은 1~3(framework·tree·flowchart)부터, v0.2에서 4~6 확장.

## 5. 파일·모듈 구성 (신규 저장소)

기존 `연구설계 챗봇`을 복제하지 않고, 공통 엔진만 가져와 새 저장소를 만든다.

```
도식화 챗봇/
├─ 소스코드/
│  ├─ app.py          # Tkinter UI (기존 셸 재사용 + 그림 미리보기 패널)
│  ├─ core.py         # API·키·파일읽기·OCR·CJK검사 (기존 거의 그대로)
│  ├─ diagram_prompts.py   # ★신규: 문서→JSON스펙 시스템 프롬프트
│  ├─ renderer.py     # ★신규: JSON스펙→SVG (공식·대칭검증·집안스타일)
│  ├─ svg_export.py   # ★신규: SVG→PNG 변환 + 저장 대화상자
│  ├─ requirements.txt
│  └─ ...
├─ 설계_도식화챗봇_v0.1_2026-07-01.md   # (이 문서)
└─ 작업메모_v0.1_YYYY-MM-DD.md
```

### core.py 재사용 범위
- 그대로: 키 저장/로드·형식검사, `read_file`/`read_file_ocr`, `check_cjk`,
  `friendly_error`, 429 재시도 로직.
- 수정: `stream_answer` → 도식 모드에서는 스트리밍 대신 **JSON 완결 응답**을 받는
  `get_spec()`로 분기(부분 JSON은 파싱 불가하므로 스트리밍 미사용).

### renderer.py 설계 원칙
- 유형별 함수: `render_framework(spec)`, `render_tree(spec)`, `render_flowchart(spec)` …
- 각 함수는 스킬의 '작성 순서'와 '대칭 검증 체크리스트'를 코드로 구현:
  1) viewBox 산정 → 2) 구역/레벨 분할 → 3) 노드 배치(공식) →
  4) 컨테이너 높이 역산 → 5) 화살표 → 6) 대칭 assert 검증.
- 스타일 상수(`PALETTE`, `FONT`, 크기)는 파일 상단에 한 곳으로 모아 집안 스타일 고정.
- 마지막에 `check_cjk(svg)` 통과 실패 시 렌더 중단·경고.

### svg_export.py
- SVG는 텍스트 저장(네이티브).
- PNG 변환: `cairosvg` 우선, 실패 시 `resvg`(단일 실행파일) 대체.
  둘 다 한글 글꼴 렌더가 관건 → 윈도우는 '맑은 고딕' 내장 경로를 명시.
- 저장은 기존 앱처럼 `asksaveasfilename`로 위치·파일명 선택.

## 6. UI 설계 (기존 챗봇 셸 확장)

```
┌───────────────────────────────────────────────┐
│ 도식화 챗봇          [모델▼] [🔑]  [ℹ️ 업데이트]   │
├──────────────────────┬────────────────────────┤
│  대화/지시 영역        │   그림 미리보기 패널        │
│  - 파일 첨부 📎        │   (SVG를 렌더링해 표시)     │
│  - "이 문서로 분석틀    │                          │
│     그려줘"           │   [SVG 저장] [PNG 저장]    │
│  - AI가 스펙 요약 표시  │   [다시 그리기] [수정지시]   │
├──────────────────────┴────────────────────────┤
│ [ 보내기 / ■ 중지 ]   읽기지원: PDF·docx·txt·md     │
└───────────────────────────────────────────────┘
```

- 왼쪽: 기존 대화형 UI 재사용(첨부·중지·모델선택·키).
- 오른쪽: **그림 미리보기 패널**(신규). Tkinter는 SVG를 직접 못 그리므로:
  - v0.1: SVG→PNG(cairosvg)로 래스터화해 `Label`에 표시(가장 단순·확실).
  - 대안: 시스템 브라우저로 미리보기(기존 앱의 HTML 내보내기와 동일 방식).
- "수정지시": 사용자가 "3번 박스 이름 바꿔"라고 하면 AI가 **JSON 스펙만 수정** →
  즉시 재렌더(전체 재생성 아님). 스펙 기반 구조의 핵심 이점.

## 7. 시스템 프롬프트(diagram_prompts.py) 핵심 설계

기존 `prompts.py`의 BASE(한자 금지·한국어 학술 문체)를 계승하되, 두뇌를 교체한다.

포함할 내용:
1. 역할: "너는 사회과학 논문 도식 설계자다. 그림을 그리지 말고 도식 설계도(JSON)만 낸다."
2. 도식 유형 판정 규칙: 문서에서 다음을 감지해 `diagram_type`을 고른다.
   - 상위개념→하위 분기 = tree / 절차·조건분기 = flowchart /
     변수 인과·매개 = path_model / 다차원 점수 비교 = radar /
     시간축 사건 흐름 = timeline / 두 축 교차·정책패키지 = framework.
3. 출력 계약: 코드블록·설명 없이 **유효한 JSON 하나만**. 스키마(4-1) 명시.
4. 내용 규율: 문서에 없는 노드·수치를 지어내지 않는다(환각 금지). 근거가 약하면
   해당 노드를 `role:"ghost"`(흐린 placeholder)로 표시하거나 사용자에게 되묻는다.
5. 라벨 규율: 한자·가나 금지, 영문 용어는 최초 1회만 '한글(영문)' 병기.
6. 분량 규율: 노드 라벨은 짧게(도식 가독성), 긴 설명은 `sub`/`caption`으로.

## 8. 기술적 난제와 대응

| 난제 | 대응 |
|------|------|
| Tkinter는 SVG 미표시 | SVG→PNG(cairosvg) 래스터화 후 Label, 또는 브라우저 미리보기 |
| PNG의 한글 글꼴 깨짐 | 렌더 시 시스템 글꼴 경로 명시(윈도우=맑은 고딕), CJK 검사 후 저장 |
| AI가 JSON 형식 이탈 | 프롬프트로 강제 + 파서에서 최초/최후 중괄호 추출 재시도 + 실패 시 1회 재요청 |
| 스펙 필드 누락 | 렌더러가 필수필드 검증 → 부족분만 사용자에게 되물음 |
| 대칭 어긋남 | 렌더러 내부 `assert`(스킬 체크리스트)로 배치 직후 자동 검증 |
| Drive 동기화 지연 | 기존 앱과 동일 주의(온라인전용 비워짐 시 실행 지연) |

## 9. 의존성(requirements 예정)

- 재사용: `openai`, `pypdf`, `python-docx`, `pymupdf`(OCR), `pynput`
- 신규: `cairosvg`(SVG→PNG) — 실패 시 `resvg` 바이너리 동봉 대안 검토
- 표준 라이브러리: `tkinter`, `json`, `xml`(SVG는 문자열 조립이라 별도 SVG 라이브러리 불필요)

## 10. 로드맵(단계)

- **v0.1 (MVP)**: framework·tree·flowchart 3종. 입력→JSON스펙→SVG→PNG 저장,
  브라우저/래스터 미리보기. 한자검사·대칭검증 내장.
- **v0.2**: path_model·radar·timeline 추가. "수정지시"로 스펙 부분수정·재렌더.
- **v0.3**: 그림 번호·캡션 자동(그림N_주제.svg 명명 규칙), 여러 도식 일괄 생성,
  논문 초안에서 "이 절을 그림으로" 지정 생성.
- **정식배포**: 기존 챗봇처럼 개인 테스트 빌드 후 동료 배포(설치안내서).
